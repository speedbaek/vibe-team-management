# 팀 업무관리 AI 툴 — 기획 아이디어 정리

---

## 1. 제품 개요

**한 줄 요약:** 팀원들이 업무 계획·진행·고민·결과를 주기적으로 기록하면, AI가 자동으로 분석·평가·코칭해주는 웹 기반 업무관리 플랫폼

**대상 팀:** 5~15명 규모, 다양한 직무(개발, 디자인, 마케팅 등)가 혼합된 팀

**핵심 가치:**
- 팀원 자율적 기록 → AI 기반 맞춤 피드백
- 관리자의 평가 부담 감소 + 객관적 데이터 기반 의사결정
- 팀원 성장을 돕는 코칭 시스템 (AI가 24시간 시니어 코치 역할)

---

## 2. 핵심 기능 구조

### 2-1. 팀원 입력 시스템 (Daily/Weekly Log)

| 주기 | 작성 항목 | 설명 |
|------|-----------|------|
| **매일** | 오늘의 할 일 | 당일 계획한 업무 목록 |
| **매일** | 완료한 일 | 실제 수행한 업무와 진행률 |
| **매일** | 고민/블로커 | 막히는 부분, 도움이 필요한 사항 |
| **매주** | 주간 회고 | 이번 주 성과, 아쉬운 점, 다음 주 방향 |
| **매주** | 목표 진행 상황 | 월간/분기 목표 대비 진행률 업데이트 |
| **수시** | 결과물 등록 | 산출물 링크, 파일, 스크린샷 등 첨부 |

**작성 UX 핵심:**
- 부담 없는 간결한 입력 폼 (5분 이내 작성 가능)
- 템플릿 제공 (직무별 맞춤 — 개발자용, 디자이너용, 마케터용 등)
- 미작성 시 리마인더 알림 (슬랙/이메일 연동)
- AI가 이전 기록 기반으로 오늘의 할 일을 미리 제안

### 2-2. 목표 관리 (OKR / Goal Tracker)

**구조:** 분기 목표(Objective) → 핵심 결과(Key Result) → 주간 액션 아이템

- 팀원이 분기 초에 자신의 OKR을 설정
- 주간 로그가 자동으로 해당 OKR에 매핑됨
- 진행률이 시각적으로 표시 (프로그레스 바, 차트)
- AI가 현재 속도 기반으로 목표 달성 가능성 예측

### 2-3. AI 분석 & 피드백 엔진

#### A. 팀원용 AI 코칭 (업무 파트너)

팀원 개개인이 AI와 1:1 대화를 통해 실시간으로 업무 도움을 받을 수 있다.

- **업무 계획 수립:** "이번 분기 OKR을 이렇게 잡았는데 현실적인가?" → AI가 이전 데이터, 업무량 고려하여 조언
- **방향성 코칭:** "A안과 B안 중 뭘 먼저 해야 할지 모르겠다" → 목표 연관성, 남은 기간 기반으로 방향 제안
- **진행 과정 모니터링:** AI가 매일 기록을 분석해 "3일간 같은 업무가 미완료 상태, 블로커가 있는지?" 능동적 질문
- **우선순위 조정:** "이 속도면 마감이 빠듯하니 우선순위를 재조정해보자" 같은 제안
- **직무별 맞춤:** 개발자에게는 기술적 조언, 마케터에게는 전략적 조언

#### B. 목표 달성률 분석
- 계획 vs 실제 수행의 갭 자동 추적
- 주간/월간 달성률 트렌드 시각화
- "이 속도라면 목표 달성까지 X주 더 필요합니다" 같은 예측 제공
- 비슷한 목표를 가진 다른 팀원 대비 진행 속도 비교 (익명)

#### C. 성과 평가 자동화
- 기록 데이터 기반으로 평가 초안 자동 생성
- 직무별 맞춤 평가 기준 (개발: 코드 품질/배포 횟수, 마케팅: 캠페인 성과/리드 수 등)
- 정량적 지표 + 정성적 기록 종합 분석
- 관리자에게 "이 팀원의 이번 분기 핵심 성과 요약" 리포트 제공

### 2-4. 데이터 공개 범위 — 계층형 권한 구조

팀원의 심리적 안전감을 확보하면서도 관리자가 필요한 맥락을 파악할 수 있도록, 기록 항목별로 공개 범위를 다르게 설정한다.

#### 🟢 관리자 자동 공개 (팀원이 기록하면 관리자에게 자동으로 보임)

| 항목 | 설명 |
|------|------|
| 업무 계획 (오늘의 할 일) | 무엇을 하려고 하는지 |
| 완료 내역 | 실제로 무엇을 했는지 |
| OKR 진행률 & 핵심 결과 수치 | 목표 대비 정량적 진척도 |
| 결과물 등록 내역 | 산출물 링크, 파일 등 |
| AI 생성 주간/월간 성과 요약 | AI가 가공한 요약 리포트 |
| 위험 신호 알림 | 진행 정체, 반복 지연 등 AI 자동 감지 |

#### 🟡 팀원 선택적 공유 (팀원이 "공유" 버튼을 눌러야 관리자에게 전달)

| 항목 | 설명 |
|------|------|
| 고민/블로커 | 공유 시 관리자에게 "도움 요청"으로 전달 |
| 주간 회고 중 "도움이 필요한 부분" | 팀원이 판단하여 선택적으로 공유 |
| 결과물에 대한 셀프 평가 코멘트 | 자기 평가를 관리자와 공유하고 싶을 때 |

#### 🔴 팀원 전용 비공개 (본인만 열람 가능, 관리자 접근 불가)

| 항목 | 설명 |
|------|------|
| AI 코칭 메시지 | 개인 맞춤 조언, 방향성 제안 |
| AI에게 한 질문 & 답변 전문 | AI와의 1:1 대화 기록 |
| 컨디션/감정 기록 (Mood) | 번아웃 추적용 개인 데이터 |
| 성장 트래킹 상세 내역 | 역량 변화 추이, 개인 성찰 기록 |

#### 관리자 역할 구조 — 복수 관리자, 범위 지정 방식

관리자는 여러 명이 될 수 있으며, 각 관리자는 본인이 담당하는 팀원의 데이터만 열람할 수 있다.

| 역할 | 권한 범위 | 예시 |
|------|-----------|------|
| **최고 관리자 (Owner)** | 전체 팀원 데이터 열람 + 관리자 지정/해제 | 대표, 본부장 |
| **관리자 (Manager)** | 담당 팀원 데이터만 열람 | 김팀장 → 개발팀 5명만 |
| **팀원 (Member)** | 본인 데이터만 | 일반 팀원 |

- 최고 관리자가 "관리자 설정" 화면에서 각 관리자의 담당 팀원을 지정
- 관리자 A는 자신이 담당하지 않는 팀원의 데이터에 접근 불가
- 한 팀원이 복수의 관리자에게 배정될 수도 있음 (예: 프로젝트 리더 + 직속 팀장)
- 관리자 본인도 팀원으로서 업무 기록을 작성할 수 있음 (관리자 겸 팀원)

#### 권한 설계 원칙

- **관리자는 "무엇을 하고 있는지"는 파악하되, "내면적 고민과 AI 대화"는 볼 수 없다**
- **관리자는 본인이 담당하는 팀원의 데이터만 열람 가능** (범위 지정 방식)
- 팀원이 솔직하게 기록할 동기를 보장하면서, 관리자의 팀 운영 가시성도 확보
- 고민 공유는 팀원 자발적 판단에 따르며, 공유 시 "도움 요청" 형태로 담당 관리자에게 전달
- AI가 비공개 데이터를 분석하더라도, 관리자에게 전달되는 리포트에는 개인 비공개 원문이 포함되지 않음 (집계/요약 수준만 반영)
- 관리자가 비공개 영역 또는 담당 외 팀원 데이터에 접근을 시도할 경우, 시스템이 차단하고 로그를 남김

### 2-5. 관리자 대시보드

- **팀 전체 현황:** 각 팀원별 목표 진행률 한눈에 보기 (🟢 자동 공개 데이터 기반)
- **위험 신호 감지:** 진행이 멈춘 업무, 반복되는 블로커, 번아웃 징후 등 자동 알림
- **도움 요청 수신함:** 팀원이 선택적으로 공유한 고민/블로커 목록 (🟡 선택 공유 데이터)
- **1:1 미팅 준비:** AI가 공개 데이터 기반으로 해당 팀원의 최근 기록 요약 + 논의 추천 주제 생성
- **팀 리포트:** 주간/월간 팀 전체 성과 요약 자동 생성

### 2-6. 팀원 개인 대시보드

- **나의 OKR 현황:** 목표별 진행률, 남은 기간, 예상 달성률
- **AI 코칭 메시지:** 맞춤 피드백과 제안사항 (🔴 본인만 열람)
- **AI 대화방:** AI에게 업무 고민을 자유롭게 질문하고 코칭 받는 공간 (🔴 본인만 열람)
- **성장 트래킹:** 월별/분기별 역량 변화 추이 (🔴 본인만 열람)
- **공유 관리:** 내 기록 중 관리자에게 공유된 항목 확인 및 공유 취소
- **동료 피드백:** 프로젝트 완료 시 상호 피드백 (선택적)

## 3. 사용자 흐름 (User Flow)

```
[팀원]
  1. "구글로 로그인" 클릭 → 개인 대시보드
  2. 매일 아침: "오늘의 계획" 작성 (AI가 어제 기록 기반으로 사전 제안)
  3. 매일 저녁: "오늘 완료한 일" 체크 + 고민 기록
  4. 수시: AI 대화방에서 업무 방향성, 우선순위 등 코칭 받기
  5. 매주 금요일: 주간 회고 작성
  6. AI 피드백 확인 & 다음 주 계획 수립

[관리자]
  1. "구글로 로그인" 클릭 → 팀 대시보드
  2. 담당 팀원별 진행 상황 확인
  3. AI가 생성한 위험 신호 / 칭찬 포인트 확인
  4. 도움 요청 수신함 확인 및 대응
  5. 1:1 미팅 전: AI 브리핑 자료 확인
  6. 분기 말: AI 성과 평가 초안 검토 & 수정
```

## 4. 기술 스택 제안

| 영역 | 추천 기술 | 이유 |
|------|-----------|------|
| **프론트엔드** | React + Next.js | SSR 지원, 빠른 개발, 풍부한 생태계 |
| **백엔드** | Node.js (Express 또는 Fastify) | 프론트와 같은 언어, 빠른 프로토타이핑 |
| **DB** | PostgreSQL + Prisma ORM | 관계형 데이터 구조에 적합, 타입 안전 |
| **인증** | NextAuth.js + Google OAuth | 구글 원클릭 로그인, 별도 회원가입 불필요 |
| **AI 엔진** | Claude API (Anthropic) | 한국어 이해 우수, 맥락 분석 능력 |
| **배포** | Vercel 또는 AWS | Next.js와 최적 호환 / 확장성 |
| **알림** | Slack API + Email (SendGrid) | 기존 워크플로우 연동 |

## 5. 데이터 모델 (핵심)

```
User (사용자)
  ├── role: owner / manager / member
  ├── department: 소속 직무
  ├── google_email: 구글 로그인 이메일
  ├── managed_users: [User]              [manager만 해당, 담당 팀원 목록]
  └── has many → Goals, DailyLogs, WeeklyReviews, AIChats

Goal (OKR 목표)                              [🟢 관리자 공개]
  ├── objective: 목표 설명
  ├── key_results: 핵심 결과 목록
  ├── quarter: 해당 분기
  ├── progress: 진행률 (0~100)
  └── has many → ActionItems

DailyLog (일일 기록)
  ├── date: 날짜
  ├── planned_tasks: 계획한 업무들          [🟢 관리자 공개]
  ├── completed_tasks: 완료한 업무들        [🟢 관리자 공개]
  ├── blockers: 고민/블로커                 [🟡 선택 공유]
  │     └── is_shared_to_manager: boolean
  ├── mood: 컨디션 (선택)                   [🔴 비공개]
  └── belongs to → User, Goal

WeeklyReview (주간 회고)
  ├── week_start: 주 시작일
  ├── achievements: 이번 주 성과            [🟢 관리자 공개]
  ├── lessons: 배운 점                      [🔴 비공개]
  ├── help_needed: 도움이 필요한 부분        [🟡 선택 공유]
  │     └── is_shared_to_manager: boolean
  ├── next_week_plan: 다음 주 계획          [🟢 관리자 공개]
  └── ai_feedback: AI 피드백 내용           [🔴 비공개]

AIChat (AI 1:1 대화)                         [🔴 비공개]
  ├── user_message: 팀원의 질문
  ├── ai_response: AI 응답
  ├── context: 관련 OKR/업무 연결
  └── belongs to → User

AIFeedback (AI 분석 결과)
  ├── type: 코칭 / 평가 / 경고
  ├── content: 피드백 내용
  ├── visibility: public / private          [코칭=🔴 | 평가요약=🟢 | 경고=🟢]
  ├── target_user: 대상 팀원
  └── created_at: 생성 시점

AccessLog (접근 로그)
  ├── accessor: 접근 시도한 사용자
  ├── target_data: 접근 대상 데이터
  ├── action: 열람 / 차단
  └── timestamp: 시점
```

## 6. AI 프롬프트 설계 방향

AI가 효과적으로 작동하려면 프롬프트 설계가 핵심입니다:

**코칭 AI의 페르소나:** 경험 많은 시니어 매니저 / 코치. 비판보다 건설적 제안 위주.

**입력 데이터:**
- 해당 팀원의 최근 2주 일일 기록
- 현재 OKR 및 진행률
- 직무 정보 (맞춤 조언을 위해)
- 이전 AI 피드백 (반복 방지)

**피드백 톤:**
- 구체적이고 실행 가능한 제안
- 잘한 점을 먼저 언급 → 개선점 제안
- 강요가 아닌 선택지 제공 ("~해보는 건 어떨까요?")

## 7. MVP 범위 (1차 프로토타입)

바이브코딩 솔로 개발 기준, 4~5주 내 완성 가능한 최소 범위.
권한은 관리자/팀원 2단계만 구현하고, 3단계(Owner/Manager/Member) 및 계층형 공개는 2차에서 추가한다.

### MVP 포함 기능 — 화면별 정리

**① 로그인 (1~2일)**
- 구글 OAuth 로그인 ("구글로 로그인" 버튼 원클릭)
- 개인 지메일(@gmail.com) 포함 모든 구글 계정 허용
- 별도 회원가입 절차 없음 — 첫 구글 로그인 시 자동 계정 생성
- 관리자가 팀원 초대 (구글 이메일 주소로 초대)
- 첫 로그인 시 역할은 기본 "팀원"으로 지정, 관리자가 권한 변경

**② 팀원 — 일일 업무 기록 (3~4일)**
- 오늘의 계획 입력
- 완료한 일 체크
- 고민/블로커 기록
- 캘린더 뷰로 과거 기록 조회

**③ 팀원 — 주간 회고 (2~3일)**
- 이번 주 성과, 배운 점, 다음 주 계획 작성
- 주간 목록 조회

**④ 팀원 — AI 코칭 대화방 (4~5일)**
- AI에게 업무 고민 자유 질문
- AI가 최근 기록 맥락을 참고하여 맞춤 답변
- 업무 계획 수립, 방향성 조언, 우선순위 제안
- 대화 기록 보관 (본인만 열람)

**⑤ 관리자 — 팀 대시보드 (4~5일)**
- 전체 팀원 목록 + 최근 기록 요약
- 팀원별 상세 기록 조회 (계획/완료 내역)
- AI가 생성한 팀원별 주간 요약 리포트
- 팀 전체 주간 요약 리포트

**⑥ 간단한 OKR (3~4일)**
- 목표 + 핵심 결과 설정
- 진행률 수동 업데이트
- 프로그레스 바 시각화

**⑦ 배포 + 마무리 (3~4일)**
- Vercel 배포, 도메인 연결
- 환경변수 분리 (추후 조직별 복제 대비)
- 기본 UI 다듬기, 버그 수정

**총 예상: 약 4~5주** (하루 2~3시간 기준)

## 8. 배포 전략 — 단일 코드베이스, 조직별 독립 배포

여러 법인/조직에 동일한 프로그램을 적용하되, 하나의 코드베이스에서 조직별로 독립된 인스턴스를 배포하는 방식을 채택한다.

### 기본 원리

```
[코드 저장소 (하나)]
  └── 동일한 소스코드
        ├── A법인 배포 → a-corp.example.com (DB-A, 설정-A)
        ├── B법인 배포 → b-corp.example.com (DB-B, 설정-B)
        └── C법인 배포 → c-corp.example.com (DB-C, 설정-C)
```

### 조직별로 달라지는 설정 (환경변수로 분리)

| 설정 항목 | 예시 |
|-----------|------|
| 조직명 / 로고 | "A법인 업무관리" |
| 도메인 주소 | a-corp.example.com |
| DB 연결 정보 | 조직별 독립 DB |
| AI API 키 | 조직별 별도 키 (비용 분리) |
| 직무 분류 | 조직 특성에 맞게 커스텀 |
| OKR 주기 설정 | 월간 / 분기 등 조직별 선택 |
| Google OAuth Client | 조직별 OAuth 설정 |
| 관리자 초기 계정 | 조직별 관리자 구글 이메일 |

### 새 조직 추가 시 작업 절차

1. 환경변수 파일(.env) 복사 후 조직 정보 수정 (5분)
2. 새 DB 생성 및 초기 세팅 (5분)
3. 배포 플랫폼(Vercel/Docker)에서 새 인스턴스 배포 (10분)
4. 도메인 연결 (5분)
5. 총 소요시간: **약 30분 이내**

### 코드 업데이트 시

- 코드 저장소에서 기능 수정 → 각 인스턴스에 재배포
- CI/CD 파이프라인 구성 시, 코드 push 한 번으로 전체 인스턴스 자동 배포 가능
- 특정 조직만 먼저 업데이트하고 나머지는 나중에 적용하는 것도 가능 (카나리 배포)

### 이 방식의 장단점

**장점:**
- 개발 복잡도가 낮음 (멀티테넌트 대비 코드가 단순)
- 조직 간 데이터가 물리적으로 완전 격리 (보안 우수)
- 한 조직에 장애가 생겨도 다른 조직에 영향 없음
- 혼자 개발·운영하기에 적합

**단점:**
- 조직이 10개 이상으로 늘어나면 배포 관리 부담 증가
- 기능 업데이트 시 각 인스턴스 재배포 필요 (CI/CD로 자동화 가능)

**향후 전환:** 조직 수가 많아져 관리가 어려워지면 그때 멀티테넌트로 전환 검토. 처음부터 환경변수로 설정을 분리해두면 전환 비용이 크지 않음.

| 팀원은 수동적 보고자 | 팀원이 AI 코칭을 통해 **자기 주도적 성장** |
| 평가는 분기 말에 한 번 | **연속적, 실시간** 피드백 |

## 10. 리스크 & 고려사항

- **작성 부담:** 매일 기록하는 것이 부담될 수 있음 → 최대한 간결한 UX, AI 사전 제안 기능으로 해소
- **데이터 프라이버시:** 계층형 권한 구조(🟢 자동공개 / 🟡 선택공유 / 🔴 비공개) 철저히 적용. AI가 비공개 데이터를 분석하더라도 관리자 리포트에는 원문 미포함. 접근 로그(AccessLog)로 비정상 접근 시도 추적
- **AI 피드백 품질:** 초기에는 피드백이 일반적일 수 있음 → 데이터 축적될수록 정밀해지도록 설계
- **조직 문화 저항:** "감시당하는 느낌" 우려 → 계층형 권한으로 비공개 영역 보장, AI 대화는 절대 관리자에게 노출되지 않음을 명시. 코칭/성장 도구임을 강조하고 팀원 자발적 참여 유도
